# 함수 본문 엣지 타입 분리 (funcbody) — 의사결정 기록

> 날짜: 2026-04-27  
> 관련 파일: `src/lib/flowchart/ast-to-flow.ts`, `src/lib/flowchart/layout.ts`, `src/components/flowchart/edges/AnimatedEdge.tsx`

---

## 문제 상황

함수 선언 예제를 순서도로 변환했을 때, 함수 본문 노드들이 메인 실행 흐름에 끼어드는 레이아웃 문제가 발생했다.

```
약속, (이름) 에게 인사하기
    이름 + "님, 안녕하세요!" 보여주기

"철수" 에게 인사하기
```

기대: 함수 본문이 함수 노드 옆에 별도로 표시  
실제: 함수 본문 노드가 메인 흐름(함수 호출 노드들) 사이에 끼어들어 순서가 뒤섞임

---

## 원인 분석

반복문 본문과 함수 본문 모두 `body` edgeType을 사용하고 있었다. dagre는 `body` 엣지를 일반 엣지와 동일하게 처리해 함수 본문 노드를 메인 흐름에 포함시켰다.

- **반복문 body**: 반복 본문 → 순차 실행 흐름의 일부 (dagre 포함이 맞음)
- **함수 본문 body**: 호출 시점에 실행되는 별도 스코프 (dagre에서 분리해야 함)

---

## 검토한 선택지

### 선택지 1: 함수 본문을 렌더링하지 않음
- 함수 노드만 표시하고 본문은 숨김
- **기각 이유**: 함수 내부 로직이 보이지 않아 디버깅 가치가 없음

### 선택지 2: 함수 본문을 별도 서브그래프 패널로 표시
- React Flow의 Group 노드 기능 활용
- **기각 이유**: 구현 복잡도가 높고 React Flow의 Group 노드는 인터랙션이 제한적

### 선택지 3: funcbody 전용 엣지 타입 도입 (채택)
- `funcbody` edgeType을 신설해 dagre에서 제외
- 함수 본문 노드를 함수 노드 왼쪽에 수동 배치
- 보라 점선 bezier로 "소속" 관계를 시각화

---

## 결정: funcbody 전용 엣지 타입 도입

### 근거

- 함수 본문은 "순차 실행 흐름"이 아닌 "선언 범위" 개념이므로 별도 처리가 의미론적으로 맞음
- 기존 코드 변경 최소화: edgeType 구분만으로 dagre/렌더링 동작을 분리할 수 있음
- 시각적으로도 점선 연결이 "이 노드들은 함수 내부"임을 직관적으로 표현

### 구현

- `ast-to-flow.ts`: DeclareFunction 본문 첫 진입 엣지에 `edgeType: 'funcbody'` 설정
- `layout.ts`: funcbody 직접 자식에서 BFS로 본문 전체 노드 수집 → 메인 dagre 제외 → 별도 dagre로 서브그래프 배치
- `AnimatedEdge.tsx`: funcbody는 보라(`#C4B5FD`) 점선 bezier로 렌더링

---

## 결과

함수 본문이 함수 노드 왼쪽에 독립적으로 표시되며, 메인 실행 흐름(함수 호출 순서)과 시각적으로 분리됨.
