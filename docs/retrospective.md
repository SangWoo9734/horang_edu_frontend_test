# 프로젝트 회고

---

## 잘된 점

- 구현 대부분을 AI가 위임받아 진행했고, 핵심 기능도 달빛약속에서 제공하는 기능을 활용했기 때문에 구현 과정에서 특별히 잘된 점을 찾기는 어렵다고 생각한다. 하지만 구현 과정과 결과를 지속적으로 검증하는 과정을 통해 큰 문제없이 구현을 완료할 수 있었고, 가장 큰 문제였던 노드 포커싱 문제도 대안을 비교하고 그중 최선의 대안을 검증하는 과정을 통해 안정적으로 해결했다는 점을 꼽고 싶다.

- 구현 이전 단계에서 요구사항 명세, 기획서, 스킬 문서 등을 상세하게 작성한 덕분에 에이전트가 일관된 방향으로 구현을 진행할 수 있었습니다. 가이드와 스펙을 충분히 준비하는 것이 에이전트 활용의 안정성에 직접적인 영향을 준다는 것을 실감했습니다.

---

## 아쉬웠던 점과 개선 방안

- 기술적으로는 마지막 실행 노드의 하이라이트 처리 부분이 아쉬움으로 남습니다. 여러 대안(`flushSync`, 직접 구독, `setTimeout` 등)을 비교 검토하고 선택한 사항이지만, 결과적으로 완전히 매끄럽지 않은 부분이 있습니다. 브라우저 렌더링 타이밍에 의존하는 방식의 한계이며, 라이브러리 수준에서 해결하거나 별도 렌더링 컨텍스트로 분리하는 방향이 근본적인 개선 방안이 될 것 같습니다.

---

## 시간이 더 있었다면 하고 싶었던 것

아이디에이션 단계에서 생각보다 많은 시간을 쓰게 됐습니다. 초기에는 초·중등 학습자를 타겟으로 달빛약속을 활용한 새로운 학습 제품을 만드는 방향으로 진행하고 싶었습니다. 아이디에이션을 진행할수록 기존 호랑에듀 제품과 방향이 겹쳐서 과제 예시 중 하나를 발전시키는 방향으로 전환했습니다.

완전히 새로운 제품을 만들기는 어렵더라도, 달빛약속이라는 언어가 가진 교육적 특성을 살려 새로운 학습 경험을 제공하는 방향은 시간이 있다면 다시 한번 시도해보고 싶습니다.
