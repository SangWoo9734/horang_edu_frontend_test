# 프로젝트 회고

---

## 잘된 점

구현 대부분을 AI가 위임받아 진행했고, 핵심 기능도 달빛약속에서 제공하는 기능을 활용했기 때문에 구현 과정에서 특별히 잘된 점을 찾기는 어렵다고 생각합니다. 하지만 구현 과정과 결과를 지속적으로 검증하는 과정을 통해 큰 문제없이 구현을 완료할 수 있었고, 가장 큰 문제였던 노드 포커싱 문제도 대안을 비교하고 그중 최선의 대안을 검증하는 과정을 통해 안정적으로 해결했다는 점을 꼽고 싶습니다.

구현 이전 단계에서 요구사항 명세, 기획서, 스킬 문서 등을 상세하게 작성한 덕분에 에이전트가 일관된 방향으로 구현을 진행할 수 있었습니다. 가이드와 스펙을 충분히 준비하는 것이 에이전트 활용의 안정성에 직접적인 영향을 준다는 것을 실감했습니다.

---

## 아쉬웠던 점, 시간이 더 있었다면 해보고 싶은 것

아이디에이션 단계에서 생각보다 많은 시간을 쓰게 됐습니다. 초기에는 초·중등 학습자를 타겟으로 달빛약속을 활용한 새로운 학습 제품을 만드는 방향으로 진행하고 싶었는데, 아이디에이션을 진행할수록 기존 호랑에듀 제품과 방향이 겹치는 느낌이 들어서, 과제 예시 중 하나를 발전시키는 방향으로 전환했습니다. 결과적으로 구현에 온전히 집중할 시간이 부족했고, 처음 기획했던 새로운 학습 경험 방향을 바꾸는 선택을 했습니다.

기술적으로는 마지막 실행 노드의 하이라이트 처리 부분이 아쉬움으로 남습니다. 여러 대안을 비교 검토하고 선택한 사항이지만, 브라우저 렌더링 타이밍에 의존하는 방식의 한계로 완전히 매끄럽지 않은 부분이 있습니다.

시간이 더 있었다면 달빛약속이라는 언어가 가진 교육적 특성을 살려 새로운 학습 경험을 제공하는 방향을 다시 한번 시도해보고 싶습니다. 구현 측면에서는 브레이크포인트나 오류 발생 지점을 더 직관적으로 보여주는 기능 등 디버거로서의 완성도를 높이는 작업도 해보고 싶습니다.

서비스 방향 측면에서도 아쉬움이 있습니다. 지금은 자유롭게 코드를 작성하고 순서도를 확인하는 형태인데, 호랑에듀의 기존 서비스처럼 문제와 함께 순서도를 제공하는 형태였다면 더 실용적인 학습 경험을 만들 수 있었을 것 같습니다.

에이전트 활용 방식에서도 모델별로 역할을 구분해보지 못한 것이 아쉽습니다. 이번에는 Claude Code 하나로 구현 전반을 진행했는데, GPT 계열은 문서 작성, Claude Code는 구현처럼 모델의 특성에 맞게 역할을 나눴다면 각 도구의 강점을 더 잘 활용할 수 있었을 것 같습니다.
