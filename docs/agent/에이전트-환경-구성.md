# 에이전트 환경 구성

## 사용한 에이전트 도구

| 도구 | 용도 |
|------|------|
| Claude (claude.ai 웹 채팅) | 아이디에이션, 기획 문서 작성, 기술 설계, 의사결정 논의 |
| Claude Code (CLI) | 전체 구현 단계 — 코드 생성, 수정, 테스트, 린트/빌드 검증 |

## 시스템 프롬프트: CLAUDE.md

프로젝트 루트의 `CLAUDE.md`가 Claude Code의 시스템 프롬프트로 작동함.

포함 내용:
- 프로젝트 개요 및 목표 (양방향 코드↔순서도 변환)
- 기술 스택 명시 (Vite 8, React, PandaCSS, React Flow, Monaco, @dalbit-yaksok/core)
- 달빛약속 API 활용 맵 (어떤 기능에 어떤 API를 써야 하는지)
- AST → 순서도 노드 타입 매핑 표
- 프로젝트 디렉토리 구조
- 디자인 토큰 (색상, 폰트)
- 코드 컨벤션 (네이밍, 타입 통일)

### 설계 원칙

CLAUDE.md에 프로젝트 전반의 맥락을 선제적으로 제공하여, 매 작업마다 컨텍스트를 반복 설명하는 비용을 줄임. 에이전트가 각 구현 결정에서 프로젝트 방향과 일관된 선택을 하도록 유도.

## 사용한 스킬

Claude Code의 커스텀 스킬(`.claude/skills/`)을 구성하여 반복 작업을 자동화:

| 스킬 | 용도 |
|------|------|
| `commit` | 변경사항을 논리 단위로 그룹화하여 Conventional Commits 형식으로 커밋 |
| `pre-commit-check` | 커밋 전 ESLint + 빌드 자동 검증 |

### commit 스킬 규칙

- 커밋 메시지에 `Co-Authored-By` 등 AI 기여자 정보 포함 금지
- `git push`는 명시적 요청 없이 실행하지 않음
- `--no-verify` 우회 금지

## MCP 서버

사용하지 않음.

## 에이전트 활용 원칙

1. **컨텍스트 선제 제공**: CLAUDE.md로 프로젝트 구조, 컨벤션, 타입 규칙을 사전 명시
2. **작업 단위 분리**: 기능별로 명확한 범위를 정하고 위임. 산출물 확인 후 다음 단계 진행
3. **검증은 직접**: 빌드/린트는 에이전트가 실행하되, 기능 동작 여부는 브라우저에서 직접 확인
4. **방향 교정**: 에이전트가 잘못된 방향(인라인 스타일, 잘못된 API 사용 등)을 제시할 때 즉시 피드백하고 원칙을 명시하여 재발 방지
5. **의사결정은 인간이**: 기술적 트레이드오프가 있는 결정(setTimeout vs flushSync, 라이브러리 패치 여부 등)은 에이전트의 분석을 참고하되 최종 판단은 직접 수행
