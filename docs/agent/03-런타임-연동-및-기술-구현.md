## 기술 설계 (초안)

### 핵심 과제: 달빛약속 코드 → 게임 캐릭터 동작

#### 방법 1: stdout 파싱
```typescript
const session = new YaksokSession({
    stdout: (msg) => {
        // "오른쪽으로 가기" → gameEngine.moveRight()
        // "줍기" → gameEngine.pickup()
        parseAndExecuteCommand(msg)
    }
})
```
- 장점: 간단, setBaseContext로 명령어를 "보여주기"로 출력하면 됨
- 단점: 문자열 파싱 의존, 에러 처리 불편

#### 방법 2: FFI 커스텀 런타임
```typescript
class GameRuntime implements Extension {
    manifest = { ffiRunner: { runtimeName: 'Game' } }
    
    async executeFFI(code, args, scope) {
        // 게임 상태 직접 조작
        return gameEngine.execute(code, args)
    }
}
```
- 장점: 타입 안전, 게임 상태 양방향 접근
- 단점: 구현 복잡도 높음

#### 방법 3: 하이브리드
- 기본 명령어는 setBaseContext + stdout으로 처리
- 복잡한 상호작용 (변수 조건 확인 등)은 FFI로 확장
- **현실적으로 가장 적합한 접근**

### 실행 흐름

```
1. 사용자가 코드 작성
2. [▶ 실행] 클릭
3. validate() → 에러 있으면 표시하고 중단
4. YaksokSession 생성 (executionDelay 설정)
5. setBaseContext로 스테이지 명령어 등록
6. runModule('main') 실행
7. runningCode 이벤트 → 에디터 줄 하이라이트
8. stdout 출력 → 게임 명령 파싱 → 캐릭터 애니메이션
9. 게임 상태 확인 → 클리어 조건 충족 여부 판단
10. 클리어 시 축하 + 다음 스테이지 안내
```
