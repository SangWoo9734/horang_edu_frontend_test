# 01. 초기 아이디에이션 및 방향 설정

## 사용한 에이전트 도구

- Claude (claude.ai 웹 채팅)

## 에이전트에게 위임한 작업의 범위

### 작업 범위
`@dalbit-yaksok/core`의 기능 문서를 기반으로, 4일 내 구현 가능한 웹앱 아이디어 브레인스토밍 및 방향성 수렴.

### 판단 근거
- 달빛약속 패키지의 기능이 다양하여 어떤 조합이 제품으로 의미 있는지 빠르게 탐색 필요
- 혼자 브레인스토밍 시 편향 위험, 에이전트와 대화하며 후보를 넓히고 좁히는 방식이 효율적

## 에이전트의 산출물 검증 방법

### 아이디어 검증 기준
1. **호랑에듀 기존 서비스와 차별화** — 이미 제공하는 기능과 겹치지 않는가
2. **달빛약속 API 활용도** — core 패키지를 얼마나 깊게 쓰는가
3. **4일 내 구현 가능성** — 핵심 기능을 완성할 수 있는가
4. **평가 기준 적합성** — 에이전트 오케스트레이션(1순위), 코드 품질(2순위), 창의성(3순위) 충족

### 검증 과정
호랑에듀 실제 서비스(app.horang.it)를 직접 탐색하여 에이전트가 제안한 아이디어와 기존 서비스의 중복 여부를 확인함.

## 에이전트 교정 이력

### 1차 교정: 타겟 사용자 가정
- 에이전트가 "코딩 입문자"를 타겟으로 가정하고 교육용 플레이그라운드 제안
- 실제로는 호랑에듀 채용 과제임을 피드백, 평가 관점으로 방향 수정

### 2차 교정: 기존 서비스 중복 (교육 플레이그라운드)
- 에이전트가 "교육용 플레이그라운드"를 추천
- 호랑에듀 웹사이트를 공유하여 유사 기능 존재 확인, 방향 전환

### 3차 교정: 퍼즐 게임 기획의 한계
- 에이전트와 퍼즐 게임(코드로 캐릭터 조작) 기획을 구체화했으나, 커스텀 명령어 학습이 달빛약속 문법 학습과 괴리가 있다는 문제 지적
- 에이전트도 동의하여 기획 재검토

### 4차 교정: 듀오링고 스타일 학습 플랫폼
- 에이전트와 함께 듀오링고 스타일 퀴즈 플랫폼(달빛루프)을 상세 기획까지 완료
- 호랑에듀 실제 서비스를 확인한 결과, 단계별 학습 + 문제풀이 + 알고리즘 문제까지 이미 제공 중
- 기획 전면 재검토

### 5차: 최종 방향 확정
- 코드 ↔ 순서도 시각화 도구(n8n 스타일)를 제안
- 호랑에듀가 하지 않는 영역이며, 과제 예시에 명시된 주제
- 평가 기준 분석 결과, 주제 창의성(3순위)보다 에이전트 활용(1순위)과 코드 품질(2순위)이 중요하므로, 리스크 낮은 명시된 주제로 확정

## 최종 결정

**달빛흐름 (Dalbit-Flow)** — 달빛약속 코드와 순서도를 실시간 변환하고, 실행 과정을 시각적으로 추적하는 도구.

### 선정 근거
- 과제 예시에 명시된 주제 → 방향성 리스크 제로
- parse(), AST, tokenize(), validate(), runningCode, executionDelay, variableSet 등 core API를 거의 전부 활용
- React Flow 라이브러리로 4일 내 완성도 있는 결과물 가능
- 호랑에듀 기존 서비스와 완전히 다른 영역

### 기각된 후보와 기각 사유

| 후보 | 기각 사유 |
|------|----------|
| 실행 시각화 플레이그라운드 | 단조로움, 창의성 부족 |
| 코드 실행 타임라인 | 단조로움 |
| 크리에이티브 캔버스 | 완성도 기준 불명확, 산으로 갈 위험 |
| 음악 플레이그라운드 | 완성도 기준 불명확 |
| 퍼즐 게임 | 커스텀 명령어 학습 ≠ 달빛약속 문법 학습 |
| 듀오링고 학습 플랫폼 | 호랑에듀 기존 서비스와 전면 중복 |
| 미니 프로젝트 빌더 | 호랑에듀 알고리즘 문제와 유사 |
