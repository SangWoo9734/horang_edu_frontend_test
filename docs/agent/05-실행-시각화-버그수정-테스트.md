# 05. 실행 엔진 + 시각화 + 버그 수정 + 테스트

## 사용한 에이전트 도구

- Claude Code (CLI)

## 에이전트에게 위임한 작업의 범위

### 작업 범위

- `runner.ts`: `YaksokSession` 연동 — stdout/stderr 캡처, `runningCode`/`variableSet` 이벤트 처리, AbortController로 실행 취소
- `ConsolePanel.tsx`, `VariablePanel.tsx`: 실행 결과/변수 현황 UI
- 실행 중 에디터 줄 하이라이트 (`deltaDecorations`)
- 실행 중 순서도 노드 하이라이트 + 진입 엣지 애니메이션
- 미연결 노드 감지 + 주황 점선 경고 표시 (`findDisconnectedNodeIds`)
- 단위 테스트 + 통합 roundtrip 테스트 (vitest) — 47개
- `outputType` 구분 (문자열 vs 변수·식) + NodeEditModal 토글 UI

### 판단 근거

- `YaksokSession` API는 문서가 명확하므로 에이전트가 이벤트 핸들러를 작성하기 적합
- 버그 수정은 원인 파악을 직접 하고, 수정 방향을 지시한 후 에이전트가 코드 반영
- 테스트는 에이전트가 작성하되, 테스트 케이스 설계(어떤 입력/출력을 검증할지)는 직접 결정

## 에이전트의 산출물을 검증한 방법

- 각 예제 코드를 실행하여 콘솔 출력, 변수 현황, 노드 하이라이트 동작 직접 확인
- 실행 중 코드 편집 시 C2F 루프가 발생하지 않는지 확인
- `npx vitest run`으로 전체 테스트 통과 확인
- 분기 내 노드 하이라이트 타이밍 이슈는 디버그 테스트를 직접 작성하여 원인 추적

## 에이전트가 잘못된 방향을 제시했을 때 교정한 방법

### 교정 1: 변수 패널 `[object Object]` 출력

에이전트가 `value.toString()`을 사용했으나 `YaksokSession`의 `ValueType`은 `.toString()`이 제대로 동작하지 않음.

- **교정**: `value.toPrint()`로 교체 지시. 라이브러리가 제공하는 올바른 직렬화 메서드 사용.

### 교정 2: 보여주기(출력) 노드 하이라이트 누락 — 핵심 버그

if/else 분기 내 마지막으로 실행되는 출력 노드가 하이라이트되지 않는 문제.

**원인 파악 과정**:
1. 에이전트가 처음에 `flushSync`를 제안 — 실제 테스트 결과 동작하지 않음
2. `runningCode` 이벤트의 line 번호와 노드의 `data.line` 매칭 여부를 디버그 테스트로 확인 → 매칭은 정확했음
3. 브라우저 렌더링 모델 분석: **브라우저는 macrotask가 끝난 후에만 페인트**함. `setExecutingNodeId('n4')`와 `setExecutingNodeId(null)` 모두 같은 macrotask 안에 있어 최종값(null)만 화면에 표시됨

**검토한 대안**: `flushSync`, 직접 구독, 라이브러리 패치, `setTimeout(executionDelay)`  
→ 상세 내용은 `docs/decisions/execution-highlight-timing.md` 참고

**최종 교정**: `finally`에서 `setTimeout(executionDelay)`로 클리어를 다음 macrotask로 지연.  
다른 노드들이 하이라이트되는 메커니즘(executionDelay setTimeout이 만드는 macrotask 경계)과 동일한 원리를 마지막 노드에도 적용.

### 교정 3: FunctionInvoke(보여주기) 구문 강조 누락

달빛약속 `보여주기` 함수 호출이 에디터에서 일반 텍스트로 표시되는 문제.

- **원인**: Monaco 테마에 `tag` 토큰 색상 규칙이 없었음
- **교정**: `dalbit-light` 테마에 `{ token: 'tag', foreground: '4F46E5', fontStyle: 'bold' }` 추가 지시
