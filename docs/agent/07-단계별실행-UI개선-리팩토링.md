# 07. 단계별 실행 + UI 개선 + 리팩토링

## 사용한 에이전트 도구

- Claude Code (CLI)

## 에이전트에게 위임한 작업의 범위

### 작업 범위

#### 단계별 실행 기능
- `YaksokSession.stepByStep = true` + `resume()` 활용한 단계별 실행 구현 (`runner.ts`)
- `execution-store`에 `isStepMode: boolean`, `status: 'stepping'` 추가
- TopNav에 [⏭ 단계별] 시작 버튼 + [⏭ 다음] 진행 버튼 추가
- 단계별 모드에서 속도 그룹 숨김, 멈춤/계속 버튼 숨김 처리

#### 헤더 UI 전면 재설계
- 속도 슬라이더 → 4단계 버튼 (느림 1500ms / 보통 700ms / 빠름 300ms / 매우빠름 80ms)
- 헤더를 속도 그룹 / 실행 제어 그룹 / 시작+상태 그룹 / 유틸 4구역으로 재구성
- 서비스명 "달빛흐름"으로 확정, 상태 칩 6종 (idle/running/paused/stepping/done/error)

#### App.tsx 리팩토링
- `App.tsx` 432줄 → 55줄: `TopNav`, `SyncBadge` → `nav/TopNav.tsx`, `CardHeader`, `SmallBtn` → `ui/CardHeader.tsx`로 분리

#### 변수 패널 UX 개선
- 값이 바뀐 변수에 650ms 노란 플래시 피드백 (`#FEF9C3` 배경)
- 패널 너비 260px → 320px 확대

#### 노드 호버 힌트
- DecisionNode: 참(초록 Left) / 거짓(빨강 Right) 레이블
- LoopNode: 반복 복귀(초록 Left) 레이블
- ProcessNode, OutputNode, FunctionNode, TerminalNode: ↑ 이전 노드 / ↓ 다음 노드 힌트

#### 컴포넌트 리팩토링 (훅 추출)
- `FlowCanvas.tsx` 271줄 → 113줄: `useFlowSync`, `useExecutionHighlight` 분리
- `NodeEditModal.tsx` 445줄 → 210줄: `useNodeForm` 훅 분리

#### 기타 버그 수정 및 개선
- NodePalette 테두리 색상 버그 수정 (CSS shorthand 충돌 → `#EEEDF8` 명시)
- CardHeader `dotColor="token(colors.primary)"` 버그 수정 → `"#706EEB"` hex 값으로 교체
- 사용 방법 모달: 최초 방문 시 자동 표시 + `localStorage` 기억
- 기능 없는 유저 배지 제거
- 테스트 47개 → 54개로 보강 (ListLoop, funcbody 엣지, while 루프 등)
- `vercel.json` 추가 — PandaCSS codegen 빌드 순서 보장

### 판단 근거

- 단계별 실행은 `YaksokSession`이 이미 `stepByStep` 프로퍼티와 `resume()` API를 제공하고 있어, 라이브러리를 수정하지 않고도 구현 가능함을 확인한 후 에이전트에게 위임
- 리팩토링 기준(200줄 이상, 훅 5개 이상)은 component-refactoring 스킬에 명시된 기준을 적용
- UI 버그는 증상을 직접 확인하고 원인(CSS shorthand, token() 함수 동작 범위) 파악 후 수정 지시

## 에이전트의 산출물을 검증한 방법

- 단계별 실행: 예제 코드를 단계별로 진행하며 노드 하이라이트 순서 및 변수 상태 변화 직접 확인
- 리팩토링: `pnpm run type-check` + `pnpm run build` 통과 여부 확인, 기존 기능 회귀 없음 확인
- 호버 힌트: 각 노드 타입별로 호버 시 레이블 정확히 표시되는지 확인
- 플래시 피드백: 실행 중 변수 갱신 시 노란 색상이 나타났다 사라지는 타이밍 확인

## 에이전트가 잘못된 방향을 제시했을 때 교정한 방법

### 교정 1: 커밋 메시지 영문 작성

에이전트가 커밋 메시지를 영어로 작성하는 실수가 반복됨.

- **발견 경위**: `git log --oneline`으로 최근 커밋 확인 시 영문 메시지 발견
- **교정**: `git filter-branch`로 21개 커밋 메시지 일괄 한글 재작성. 이후 memory에 규칙 저장

### 교정 2: funcbody 엣지 — 함수 본문 레이아웃 오류

함수 선언 노드의 본문 노드들이 메인 흐름과 연결된 것처럼 dagre가 배치하는 문제.

- **원인 파악**: `body` edgeType을 반복문과 함수 선언이 공유하고 있어 dagre가 함수 본문 노드를 메인 흐름에 포함시킴
- **교정**: 함수 본문 전용 `funcbody` edgeType 도입 → dagre에서 제외 → 함수 노드 왼쪽에 수동 배치 → `AnimatedEdge`에서 보라 점선으로 "소속" 관계 시각화

### 교정 3: `token()` 함수를 인라인 스타일에서 사용

에이전트가 `dotColor="token(colors.primary)"` 형태로 PandaCSS 토큰을 인라인 스타일에 전달.

- **원인**: `token()`은 CSS 함수로, `css()` 내부에서만 동작하며 JS 문자열로 전달 시 리터럴 문자열로 처리됨
- **교정**: `"#706EEB"` hex 값으로 교체
