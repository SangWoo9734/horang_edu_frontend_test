# 02. 프로젝트 셋업 + 레이아웃 + 에디터

## 사용한 에이전트 도구

- Claude Code (CLI)

## 에이전트에게 위임한 작업의 범위

### 작업 범위

- Vite + React + PandaCSS + TypeScript 프로젝트 초기 설정
- Zustand 스토어 4개 골격 (`editor`, `flowchart`, `execution`, `ui`)
- 4분할 패널 레이아웃 (`layout.tsx`) + 헤더 컨트롤 바
- Monaco 에디터 마운트 + 달빛약속 언어 등록 (`DalbitYaksokApplier`)
- `FlowNodeData`, `ExecutionStatus` 공통 타입 정의

### 판단 근거

- 보일러플레이트 설정과 반복적인 스캐폴딩은 에이전트에게 위임하여 설계에 집중
- 스토어 구조는 상태 흐름을 미리 설계하고, 에이전트에게 구현만 위임 (4개 스토어 분리 방식은 사전에 직접 결정)
- Monaco 언어 등록은 `@dalbit-yaksok/monaco-language-provider` API를 따르므로 에이전트가 문서 기반으로 처리하기 적합

## 에이전트의 산출물을 검증한 방법

- `pnpm dev`로 개발 서버 실행 후 레이아웃 시각적 확인
- Monaco 에디터에 달빛약속 코드 입력 → 구문 강조 동작 여부 직접 확인
- 스토어 타입이 설계한 인터페이스와 일치하는지 코드 리뷰
- `npm run build`로 타입 에러 없음 확인

## 에이전트가 잘못된 방향을 제시했을 때 교정한 방법

해당 없음. 이 단계는 명확한 스펙이 존재하는 설정 작업이었으므로 방향 교정 없이 진행.
