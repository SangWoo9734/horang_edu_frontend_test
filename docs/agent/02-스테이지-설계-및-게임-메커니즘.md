## 게임 메커니즘 설계 (초안)

### 기본 구조
- **그리드**: 5x5 ~ 7x7 타일맵
- **캐릭터**: 플레이어가 코드로 조작하는 주인공
- **목표**: 스테이지별 미션 클리어 (목적지 도달, 아이템 수집 등)
- **입력**: 달빛약속 코드 작성 → 실행 버튼
- **출력**: 캐릭터가 코드에 따라 이동/동작 (애니메이션)

### 타일 종류 (초안)
| 타일 | 설명 |
|------|------|
| 🟩 바닥 | 이동 가능 |
| 🧱 벽 | 이동 불가 |
| ⭐ 목표 | 도달하면 클리어 |
| 🔑 열쇠 | 수집 아이템 |
| 🚪 문 | 열쇠가 있어야 통과 |
| 💎 보석 | 보너스 아이템 |

### 기본 명령어 (setBaseContext로 등록)
```
위로 가기
아래로 가기
왼쪽으로 가기
오른쪽으로 가기
줍기
사용하기
```

### 스테이지 설계

#### 스테이지 1: 순차 실행 — "첫 걸음"
- **학습 목표**: `위로 가기`, `오른쪽으로 가기` 등 기본 명령어 순차 실행
- **맵**: 5x5, 직선 경로
- **미션**: 시작점 → 목표 지점 도달
- **예시 정답**:
  ```
  오른쪽으로 가기
  오른쪽으로 가기
  위로 가기
  ```

#### 스테이지 2: 변수 — "열쇠와 문"
- **학습 목표**: 변수에 값 저장하기
- **맵**: 5x5, 열쇠와 잠긴 문
- **미션**: 비밀번호를 변수에 저장하고 문 열기
- **힌트**: 맵에 비밀번호 숫자가 표시됨
- **예시 정답**:
  ```
  오른쪽으로 가기
  비밀번호 = 1234
  문 열기 (비밀번호)
  위로 가기
  ```

#### 스테이지 3: 반복문 — "긴 복도"
- **학습 목표**: `반복` 문법으로 반복 동작 줄이기
- **맵**: 7x7, 긴 직선 복도
- **미션**: 10칸 이동해야 하는데 코드 줄 수 제한 있음
- **예시 정답**:
  ```
  반복 10번
      오른쪽으로 가기
  ```

#### 스테이지 4: 조건문 — "갈림길"
- **학습 목표**: `만약 ~ 이면` 분기 처리
- **맵**: 5x5, 갈림길 + 랜덤 요소
- **미션**: 조건에 따라 다른 경로 선택
- **예시 정답**:
  ```
  만약 열쇠가 있으면
      왼쪽으로 가기
  아니면
      오른쪽으로 가기
  ```

#### 스테이지 5: 함수 — "반복 패턴"
- **학습 목표**: `약속` (함수) 정의와 호출
- **맵**: 7x7, 같은 패턴이 반복되는 미로
- **미션**: 반복 패턴을 함수로 정의해서 효율적으로 클리어
- **예시 정답**:
  ```
  약속, ㄱ자 이동하기
      오른쪽으로 가기
      오른쪽으로 가기
      위로 가기

  ㄱ자 이동하기
  ㄱ자 이동하기
  ㄱ자 이동하기
  ```
