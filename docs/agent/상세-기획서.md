# 달빛약속 코드 ↔ 순서도 시각화 도구 — 상세 기획서

## 01. 프로젝트 개요

- **서비스 명칭**: 달빛흐름 (Dalbit-Flow)
- **한 줄 정의**: 달빛약속 코드와 순서도를 양방향으로 실시간 변환하고, 실행 과정을 시각적으로 추적하는 도구
- **핵심 가치**: 코드의 논리 구조를 시각적으로 이해할 수 있게 하여, "이 코드가 어떤 흐름으로 실행되는지"를 직관적으로 보여줌

### 타겟 사용자

호랑에듀에서 한글 코딩(달빛약속)을 배우는 학생

### 해결하는 문제

1. **"코드는 작성했는데 왜 이렇게 동작하는지 모르겠어요"** — 코드를 순서도로 변환하여 실행 흐름을 시각적으로 추적. 어떤 분기를 탔는지, 반복이 몇 번 돌았는지 눈으로 확인 가능.
2. **"어떤 구조로 코드를 짜야 할지 막막해요"** — 순서도로 논리를 먼저 설계하고, 그걸 코드로 자동 변환. "이런 흐름이 코드로는 이렇게 되는구나"를 체험.

### 호랑에듀 서비스와의 연결

호랑에듀의 기존 학습 파이프라인:

```
개념 학습 → 코드 작성 → 실행 결과 확인 → (여기가 비어있음) → 더 복잡한 코드 도전
```

달빛흐름이 채우는 자리:

```
개념 학습 → 코드 작성 → 실행 결과 확인 → [달빛흐름: 코드 구조 이해] → 더 복잡한 코드 도전
```

호랑에듀는 "코드 실행 결과"를 보여주지만, "코드가 어떤 흐름으로 실행되는지"를 보여주는 도구는 없음. 달빛흐름은 이 간극을 메워서, 학생이 코드의 결과뿐 아니라 과정까지 이해할 수 있게 도움.

## 02. 핵심 기능

### 기능 1: 코드 → 순서도 변환 (필수)

- 왼쪽 에디터에 달빛약속 코드를 작성하면, 오른쪽에 순서도가 실시간 생성
- AST 노드 타입별로 순서도 노드 매핑
- `parseOptimistically()`로 타이핑 중에도 가능한 범위까지 순서도 업데이트

### 기능 2: 실행 흐름 시각화 (필수)

- 코드 실행 시 `runningCode` 이벤트로 현재 실행 중인 노드를 순서도에서 하이라이트
- `executionDelay`로 실행 속도 조절 (슬라이더)
- `variableSet` 이벤트로 변수 상태 패널 실시간 업데이트
- 실행 경로가 순서도 위에 애니메이션으로 표시

### 기능 3: 코드 에디터 통합 (필수)

- Monaco Editor + 달빛약속 언어 지원
- `validate()`로 실시간 에러 표시
- `runningCode` 이벤트로 현재 실행 줄 하이라이트

### 기능 4: 순서도 → 코드 생성 (필수 — 양방향)

- 순서도 캔버스에서 노드를 추가/연결하면 달빛약속 코드가 자동 생성
- 코드 에디터에 실시간 반영 (양방향 동기화)
- **지원 노드 타입 (MVP 범위)**:
  - 처리 노드 추가 (변수 할당: `이름 = 값`)
  - 출력 노드 추가 (`"내용" 보여주기`)
  - 분기 노드 추가 (조건문: `만약 ~ 이면 / 아니면`)
  - 반복 노드 추가 (`반복 N번`)
  - 노드 간 연결 (실행 순서 정의)
  - 노드 삭제
- **노드 추가 방식**: 순서도 캔버스에 팔레트/툴바에서 드래그하여 배치, 또는 우클릭 메뉴
- **코드 생성 로직**: 순서도 노드 그래프를 위상 정렬 → 들여쓰기 포함 달빛약속 코드 문자열로 직렬화
- **동기화 전략**: 코드 편집 시 → 순서도 업데이트 (parse), 순서도 편집 시 → 코드 업데이트 (직렬화). 충돌 방지를 위해 마지막 편집 소스를 추적

## 05. AST → 순서도 노드 매핑 규칙

### 노드 타입별 순서도 표현

| AST 노드                  | 순서도 모양 | 색상   | 표시 내용     |
| ------------------------- | ----------- | ------ | ------------- |
| Block (시작)              | 둥근 사각형 | 회색   | "시작"        |
| Block (끝)                | 둥근 사각형 | 회색   | "끝"          |
| SetVariable               | 사각형      | 파랑   | `변수명 = 값` |
| IfStatement               | 마름모      | 노랑   | 조건식        |
| CountLoop                 | 육각형      | 초록   | `N번 반복`    |
| ConditionalLoop           | 육각형      | 초록   | `조건 동안`   |
| ListLoop                  | 육각형      | 초록   | `리스트 반복` |
| FunctionInvoke (보여주기) | 평행사변형  | 보라   | 출력 내용     |
| FunctionInvoke (기타)     | 이중 사각형 | 보라   | 함수 호출     |
| DeclareFunction           | 서브그래프  | 연보라 | 함수 이름     |
| Return                    | 둥근 사각형 | 빨강   | "반환"        |

### 엣지 규칙

| 상황        | 라벨   | 스타일      |
| ----------- | ------ | ----------- |
| 순차 실행   | (없음) | 실선 화살표 |
| 조건문 참   | "참"   | 실선, 초록  |
| 조건문 거짓 | "거짓" | 점선, 빨강  |
| 반복 복귀   | (없음) | 점선 (위로) |

## 05-2. 순서도 → 코드 생성 규칙 (역방향)

### 노드 추가 시 입력 폼

| 노드 타입     | 사용자 입력 | 생성되는 코드              |
| ------------- | ----------- | -------------------------- |
| 처리 (변수)   | 변수명, 값  | `변수명 = 값`              |
| 출력          | 출력 내용   | `"내용" 보여주기`          |
| 분기 (조건문) | 조건식      | `만약 조건이면` / `아니면` |
| 반복          | 반복 횟수   | `반복 N번`                 |

### 코드 생성 알고리즘

```
function flowToCode(nodes, edges):
    1. 시작 노드 찾기
    2. 엣지를 따라 위상 정렬 (topological sort)
    3. 각 노드를 순서대로 방문하며 코드 생성:
        - 처리/출력: 해당 줄 코드 추가
        - 분기: "만약 조건이면" 출력
            - 참 엣지를 따라 하위 노드 코드 생성 (들여쓰기 +1)
            - "아니면" 출력 (거짓 엣지가 있으면)
            - 거짓 엣지 하위 노드 코드 생성 (들여쓰기 +1)
        - 반복: "반복 N번" 출력
            - 반복 내부 엣지를 따라 하위 노드 코드 생성 (들여쓰기 +1)
    4. 들여쓰기 적용하여 최종 코드 문자열 반환
```

### 양방향 동기화 전략

```
상태: lastEditSource = 'code' | 'flowchart'

코드 편집 시:
    lastEditSource = 'code'
    parse(code) → AST → 순서도 노드 업데이트
    (순서도 → 코드 방향은 무시)

순서도 편집 시:
    lastEditSource = 'flowchart'
    flowToCode(nodes, edges) → 코드 업데이트
    (코드 → 순서도 방향은 무시, 이미 순서도가 소스)

debounce 적용: 양쪽 모두 300ms
```

### 노드 추가 UI

순서도 캔버스 좌측에 노드 팔레트(툴바):

```
┌──────┐
│ □ 처리│  ← 드래그하여 캔버스에 배치
│ ◇ 분기│
│ ⬡ 반복│
│ ▱ 출력│
└──────┘
```

- 팔레트에서 드래그하여 캔버스에 드롭 → 입력 폼 팝업 → 값 입력 → 노드 생성
- 노드 더블클릭 → 값 수정 가능
- 노드 간 핸들을 드래그하여 연결 → 엣지 생성

### 인터랙션

- 코드 변경 → 순서도 실시간 업데이트 (debounce)
- 실행 → 순서도 노드 + 에디터 줄 동시 하이라이트
- 속도 슬라이더 → executionDelay 조절
- 순서도 노드 클릭 → 에디터 해당 줄로 이동
- 예제 드롭다운 → 프리셋 코드 로드

## 11. 의사결정 로그

| 항목       | 결정                 | 근거                                                                                                                          |
| ---------- | -------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| 주제       | 코드 ↔ 순서도 시각화 | 과제 예시 명시, API 최다 활용, 기존 서비스 비중복                                                                             |
| 프레임워크 | React SPA (Vite)     | SSR 불필요, 클라이언트 전용                                                                                                   |
| 순서도     | React Flow           | 커스텀 노드, 생태계                                                                                                           |
| 배치       | dagre                | DAG 레이아웃 적합                                                                                                             |
| 양방향     | MVP에 포함 (D-3)     | 단방향만으로는 "AST 뷰어"에 불과, 양방향이 핵심 차별점. 노드 타입 4개(처리/출력/분기/반복)로 스코프 제한하여 1일 내 구현 가능 |
| 레이아웃   | 데스크탑 전용        | 에디터+순서도 동시 표시                                                                                                       |
| 테마       | 다크 모드 단일       | 코드/시각화 적합                                                                                                              |
| 상태관리   | Zustand 4개 스토어   | 관심사 분리                                                                                                                   |
